
# 表达式计算器增强功能开发任务书

## 一、目标概述

在现有 `ExpressionCalculator` 基础上扩展以下能力：

| 功能 | 描述 | 示例 |
|------|------|------|
| **Map / List / 数组下标访问** | 使用 `[key]` 语法访问集合元素，`key` 可以是字面量、变量或表达式 | `map['name']`<br>`list[0]`<br>`array[index + 1]` |
| **精确数字类型** | 算术运算结果保留最小必要类型，`int + int` 返回 `Integer`，而非总是 `BigDecimal` | `1 + 2` → `Integer(3)` |
| **类型强制转换** | 支持 `(type) expr` 语法，将表达式结果转换为目标类型 | `(int)(10 + 10L)` → `int` |
| **可扩展的转换器注册** | 提供 SPI 风格的转换器注册表，允许用户自定义类型转换逻辑 | `ConverterRegistry.register("int", v -> ...)` |

## 二、实现思路与架构调整

### 2.1 总体设计原则

- **向后兼容**：不破坏现有 `calculation` / `compareCalculation` / `evaluate` 接口行为。
- **最小侵入**：仅在解析层（词法/语法）增加对新语法的支持，运行时语义层通过 `Operator` 扩展实现。
- **可扩展性**：下标访问运算符 `[]` 注册为二元运算符，类型转换注册为一元运算符 `(type)`。

### 2.2 语法扩展定义

#### 2.2.1 下标访问运算符 `[]`

- 语法：`primary [ expression ]`
- 优先级：与字段访问 `.` 同级（最高优先级）。
- 结合性：左结合。
- 实现方式：在 `OperatorRegistry` 中注册一个特殊二元运算符 `"[]"`，其 `apply` 逻辑根据左操作数类型分发：
    - `Map` → `map.get(key)`
    - `List` → `list.get(index)`（需将 key 转为 `int`）
    - 数组 → `Array.get(array, index)`
    - 字符串 → `str.charAt(index)`（可选）

#### 2.2.2 类型转换运算符 `(type)`

- 语法：`( identifier ) unaryExpression`
- 优先级：与一元运算符同级（`PRECEDENCE_UNARY`）。
- 实现方式：在解析阶段识别 `(` 后紧跟类型名再跟 `)` 的模式，将其视为一元运算符，注册到 `OperatorRegistry`。
- 类型名允许：`int`、`long`、`double`、`float`、`short`、`byte`、`char`、`boolean`、`String`、`BigDecimal`，以及用户自定义扩展名。

#### 2.2.3 数字类型精度策略

修改 `ExpressionRuntimeSupport` 中的算术运算方法（`add`、`subtract` 等），使其根据操作数类型决定返回类型：

- 若两个操作数均为整数类型（`Byte`、`Short`、`Integer`、`Long`），且运算未溢出，返回对应宽化类型的整数（如 `int + int` → `int`，`int + long` → `long`）。
- 若任一操作数为浮点类型（`Float`、`Double`）或 `BigDecimal`，则返回 `BigDecimal`（保证精度）。

### 2.3 新增组件清单

| 组件 | 职责 |
|------|------|
| `IndexOperator` | 实现 `[]` 下标访问逻辑 |
| `CastOperator` | 实现 `(type)` 类型转换逻辑 |
| `TypeConverter` 接口 | 定义类型转换函数式接口 |
| `ConverterRegistry` | 管理类型转换器的注册与查找 |
| `DefaultConverters` | 提供内置转换器（int、long、String 等） |
| `NumericArithmetic` | 处理整数精度保持的算术逻辑 |

## 三、分阶段实现步骤

### Phase 1：精确数字类型

**任务：** 修改 `ExpressionRuntimeSupport` 中的 `add`、`subtract`、`multiply`、`divide` 方法。

**要求：**
1. 检测左右操作数是否均为整数类型（`Byte`、`Short`、`Integer`、`Long`）。
2. 若是，则尝试用 `long` 进行运算，若结果在 `int` 范围内则返回 `Integer`，否则返回 `Long`（根据操作数范围进一步缩小）。
3. 若包含浮点或 `BigDecimal`，回退到现有 `BigDecimal` 逻辑。

**验收要点：**
- `1 + 2` → `Integer(3)`
- `1 + 2L` → `Long(3)`
- `1 + 2.0` → `BigDecimal(3)`
- `Integer.MAX_VALUE + 1` → `Long(2147483648)`

---

### Phase 2：下标访问运算符 `[]`

**任务：** 在解析器中增加对 `[` 的识别，并注册 `[]` 二元运算符。

**步骤：**
1. 在 `OperatorRegistry` 中注册 `"[]"` 二元运算符，优先级设为 `PRECEDENCE_PRIMARY`（最高，例如 100）。
2. 修改 `IterativeExpressionEngine.Evaluator`：
    - 当 `expectOperand == false` 且遇到 `[` 时，视为二元运算符，解析其后的表达式直到匹配的 `]`。
3. 实现 `IndexOperator` 的 `apply` 方法，处理 `Map`、`List`、数组和字符串。

**验收要点：**
- `map['key']` → 对应值
- `list[0]` → 第一个元素
- `array[index]` → 动态索引
- `"hello"[1]` → `'e'`
- 支持嵌套：`map[list[0]]`

---

### Phase 3：类型转换运算符 `(type)`

**任务：** 支持 `(int) expr` 语法。

**步骤：**
1. 创建 `ConverterRegistry`，提供 `register(String name, TypeConverter converter)` 和 `convert(String name, Object value)` 方法。
2. 预注册 `int`、`long`、`double`、`String` 等标准转换器。
3. 在 `IterativeExpressionEngine.Evaluator.parsePrefixExpression()` 中：
    - 检查当前位置是否为 `(`，且后续紧跟标识符和 `)`，且标识符在 `ConverterRegistry` 中存在。
    - 若是，则消费 `(type)`，将剩余部分作为一元运算符 `cast(type)` 处理。
4. 实现 `CastOperator`，调用 `ConverterRegistry` 完成转换。

**验收要点：**
- `(int)(10 + 10L)` → `Integer(20)`
- `(String)123` → `"123"`
- 用户自定义转换器 `myType` 生效。

## 五、注意事项与约束

- **Java 8 兼容**：不使用 `var`、模式匹配、`List.of` 等。

- **性能考量**：避免在热路径中频繁反射；`ConverterRegistry` 使用 `ConcurrentHashMap` 缓存。
- **文档注释**：为新类、新方法编写详尽的 JavaDoc，说明语法和示例。
